文章正文里禁止出现以下类型的表述：

1. **预告下一篇**：如"下一篇（010）我们拆 Devin"、"下周继续聊 XXX"
2. **引用上一篇**：如"上一篇（008）我们拆了 Claude Code"、"我们在 008 里讲过"
3. **声明系列位置**：如"这是《Agent 架构剖析》系列第二篇"、"本系列第 N 篇"
4. **假设读者读过前文**：如"我们讲过"、"回忆一下上次"

**Why:** 2026-04-24 写 009 Hermes Agent 时，用户明确说"文章里不要预测下一篇，也不要描述第一篇以及和第几篇的关系，我发布不一定严格按照这个来"。公众号文章实际发布顺序/时间可能和 article_log 里的编号不一致，写死了顺序关系会让文章之间互相"穿帮"。

**How to apply:**
- 每篇文章都要能独立读，不依赖前后文
- 可以横向对比同类产品（如 Claude Code vs Hermes），但只用产品名，不用"上一篇"之类的顺序词
- 结尾不留"下期预告"，改用收束性的总结或开放式提问
- frontmatter 里的 `series: xxx · 02` 这类元数据可以保留（不会渲染到正文），但正文禁止提"· 02"
- 横向产品对比里自然提到其他 Agent 的架构特征是允许的，只要不绑定"我们第 X 篇讲过"这种顺序锚点
