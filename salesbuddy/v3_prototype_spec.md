**/Users/tangcaocao/Downloads/sales_buddy_v3.html** 由业务（说话人 3）本人用 Claude 输出，是销售端功能的**产品规格事实源**。后续讨论直接拿 v3 当基线，不需要每次重新对齐交互。

## 总体结构

- 左侧栏 3 段：工作台 / 管理（即将上线）/ 工具（即将上线）——已去掉角色切换，销售端单角色
- 工作台 4 Tab：工作台首页 / 客户看板 / 业绩看板 / 地图打卡
- **底部持久 AI Bar**：4 个快捷 chip（今日重点 / 东莞下一步 / ROI 话术 / 竞品应对），取代 v2 的右侧抽屉
- **右侧滑出客户详情 panel**：3 Tab（AI 录音解析 / 客户档案 / 跟进记录）
- 录音上传入口：顶部持久按钮，支持 文件 + 飞书妙记 URL + 飞书文档链接
- **"→ 转 CRM"** 按钮：客户从 salesbuddy 档案推送到正式 CRM 的交接动作

## 5 个 AI 分析模块字段 schema

1. **下一步建议**：三级标签 **必做 / 重要 / 加分**（业务语言，不要 P0/P1/P2）
2. **意向识别**：四层 = 等级（A/B/C）+ 伪意向风险 + 正向信号 + 隐性风险
3. **质量评估**：5 维度评分 + "最大短板"叙述
   - 开场 20 分 / 探需 25 / 价值 25 / 推进 20 / 节奏 10
4. **SPICED**：6 行覆盖率表（详见 spiced_framework.md）
5. **金牌话术卡**：标签（行业/场景/阶段）+ 引文 + 逻辑说明 + 复用指引 + 一键复制

## 其他关键决策

- **多录音合并**：客户是生命周期实体，增量合并而非"最新覆盖"
- **类目体系**：按**有赞 CRM 类目**为准（资产驱动：话术/资料/客保都挂在这棵树上）；复合行业可能需要"主类目+辅助标签"
- **通话阶段标签**：AI 自动打"初访/深挖/报价/签约"等阶段
- **预估金额+概率**：AI 给出置信度而非单点估值
- **AI 推理链**：关键判断展示依据引文

**Why:** 业务已经投入时间自己做 demo，后续对齐成本最低的方式是以 v3 为锚
**How to apply:** 接到需求先对 v3 里对应区块；v3 没覆盖的再单独讨论；若要偏离 v3 必须显式告知业务并说明原因
